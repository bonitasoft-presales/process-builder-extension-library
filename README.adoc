A reusable Java 17 library that centralizes enums, constants, records, validators, utilities and the REST execution engine shared across the *Process Builder* ecosystem: actor filters, connectors, REST API extensions and process diagrams.

== 1. What It Does

Presales and customer projects routinely duplicate the same building blocks: JSON validation, BDM audit utilities, identity and membership lookups, REST authentication (7 schemes including OAuth2 JWT Bearer), template substitution, multipart file uploads, email recipient resolution, password cryptography, GZIP compression and exception translation. This library consolidates them behind a stable, versioned contract so that every downstream project -- in particular link:https://github.com/bonitasoft-presales/bwh-rest-execution-connector[bwh-rest-execution-connector] -- consumes a single, tested source of truth.

== 2. Package Catalog

[cols="1,3",options="header"]
|===
| Package | Purpose

| `constants`
| Immutable constants: `Constants` (Bonita profile names, common strings), `SchemaConstants` (OpenAPI resource path, schema prefix).

| `enums` (26 enums)
| Domain enums. Highlights: `RestAuthenticationType` (NONE, BASIC, BEARER, API_KEY, OAUTH2_CLIENT_CREDENTIALS, OAUTH2_PASSWORD, *OAUTH2_JWT_BEARER*, DIGEST, NTLM, CERTIFICATE, CUSTOM), `RestHttpMethod` (GET, POST, PUT, PATCH, DELETE, HEAD, OPTIONS -- each with a `supportsBody` flag), `RestContentType` (JSON, XML, TEXT_PLAIN, FORM_URLENCODED, MULTIPART_FORM_DATA, BINARY, PDF -- each with its MIME string), `RestApiKeyLocation`, `RestOAuth2ClientAuthMethod`, `ActionType`, `ProcessStateType`, `CriticalityType`, `SmtpType`, and more.

| `records`
| Immutable Java 17 records. `RestAuthConfig` (sealed interface with `NoAuth`, `BasicAuth`, `BearerAuth`, `ApiKeyAuth`, `OAuth2ClientCredentials`, `OAuth2Password`, `OAuth2JwtBearer`, `CustomAuth` implementations), `RestServiceRequest` (fluent builder), `RestServiceResponse` (body parsing, status checks, JSON field extraction), `UserRecord`, `UserList`, `InvolvedUsersData`, `StepFieldRef`, `LoadedSchema`, `UsersConfigRecord`.

| `execution`
| REST execution engine -- the backbone of the BWH REST Execution Connector. `ConnectorExecutionEngine` (orchestrates the full request lifecycle), `ConnectorRequest` / `ConnectorResponse` (data contracts), `HttpExecutor` (HTTP/1.1 client with connection pooling, OAuth2 token caching, Bonita session caching), `AuthPipeline` (normalize -> decrypt -> parse auth config), `TemplateSubstitution` (`++{{placeholder}}++` resolution), `MultipartRelatedBuilder` (RFC 2387 multipart body for Google Drive, Gmail, MS Graph file uploads).

| `extension` (24 utilities)
| Bonita-facing helpers: `BDMAuditUtils`, `ConfigurationUtils`, `EmailRecipientsHelper`, `ExceptionUtils`, `ExecuteSqlDatasource`, `IdentityUtils`, `InputValidationUtils`, `InvolvedUsersParser`, `JsonNodeUtils` (comparison operators, safe navigation), `JsonValidationUtils`, `MembershipUtils`, `PBHtmlUtils`, `PBStringUtils` (template variable resolution), `PasswordCrypto` (AES/GCM encryption used by `AuthPipeline`), `ProcessInstance`, `ProcessUtils`, `ProfileUtis`, `QueryParamValidator`, `RestServiceConstants`, `StorageUtils`, `TaskAssignationUtils`, `TemplateDataResolver`, `Utils` (timing), `ValidationException`.

| `extension.template`
| REST API template system: `RestApiTemplate` (record with methods, auth, headers), `AuthConfig` (sealed interface: `NoAuthConfig`, `BasicAuthConfig`, `BearerAuthConfig`, `ApiKeyAuthConfig`, `OAuth2ClientConfig`, `OAuth2PasswordConfig`), `RestApiCatalog`.

| `mapper`
| `UserMapper` -- converts Bonita `User` lists to `Long` ID lists.

| `validation`
| `JsonSchemaValidator` (validates JSON against OpenAPI 3 schemas), `SchemaResolver` (loads and resolves `$ref` from `openapi.yaml`), `ProcessOptionValidator`.

| `utils`
| `CompressionUtils` -- GZIP + Base64 encoding/decoding (70-90% size reduction for JSON payloads).
|===

== 4. Consuming the Library

=== 4.1. Maven coordinates

[source,xml,subs="+attributes"]
----
<dependency>
    <groupId>com.bonitasoft.processbuilder.extension</groupId>
    <artifactId>process-builder-extension-library</artifactId>
    <version>{artifact-version}</version>
</dependency>
----

=== 4.2. GitHub Packages authentication

Add to `~/.m2/settings.xml` (PAT with `read:packages` scope):

[source,xml]
----
<servers>
    <server>
        <id>github</id>
        <username>YOUR_GITHUB_USERNAME</username>
        <password>YOUR_GITHUB_TOKEN</password>
    </server>
</servers>
----

Declare the repository in your POM:

[source,xml]
----
<repositories>
    <repository>
        <id>github</id>
        <url>https://maven.pkg.github.com/bonitasoft-presales/process-builder-extension-library</url>
    </repository>
</repositories>
----

=== 4.3. Local JAR in Bonita Studio

Import `target/process-builder-extension-library-<version>.jar` as a project dependency. See link:https://documentation.bonitasoft.com/bonita/latest/managing-extension-studio[managing Studio extensions].

== 5. Quick Start

=== 5.1. Execute a configuration-driven REST call

[source,java]
----
ConnectorRequest  request  = ConnectorRequest.builder(configJson)
        .params(Map.of("userId", "42"))
        .timeoutMs(15000)
        .verifySsl(true)
        .build();

ConnectorResponse response = new ConnectorExecutionEngine().execute(request);

if (response.success()) {
    System.out.println(response.statusCode());   // 200
    System.out.println(response.responseBody());  // {"name":"..."}
}
----

=== 5.2. Build a typed REST request directly

[source,java]
----
RestServiceRequest request = RestServiceRequest.builder("https://api.example.com/users")
        .method(RestHttpMethod.GET)
        .auth(RestAuthConfig.bearer("eyJhbGci..."))
        .header("Accept", "application/json")
        .timeoutMs(10000)
        .build();

RestServiceResponse response = new HttpExecutor().execute(request);

if (response.isSuccessful()) {
    Optional<JsonNode> json = response.bodyAsJson();
}
----

=== 5.3. Validate JSON against OpenAPI schema

[source,java]
----
boolean valid = JsonSchemaValidator.isJsonValidForType("INSERT", "Parameter", jsonString);
----

=== 5.4. Compress large JSON payloads

[source,java]
----
String compressed   = CompressionUtils.compress(largeJson);   // GZIP + Base64
String decompressed = CompressionUtils.decompress(compressed);
----

== 8. Testing

Every class requires *two* test files:

- `MyClassTest.java` -- JUnit 5 + AssertJ + Mockito unit tests.
- `MyClassPropertyTest.java` -- jqwik `@Property` tests for invariants.

*Coverage targets:* 95%+ line/branch, 85%+ mutation (PIT). Goal is 100%/95%.

Tests run in parallel (JUnit Jupiter concurrent mode). Current test count: *149 test files, ~3700 tests*.

== 10. Troubleshooting

[cols="1,2,2",options="header"]
|===
| Symptom | Cause | Resolution

| `401` resolving dependency
| Wrong GitHub token.
| PAT with `read:packages` in `settings.xml`.

| `NoClassDefFoundError: jackson`
| Jackson is `provided`.
| Declare on consumer's runtime classpath (Bonita bundles it).

| `InaccessibleObjectException`
| JDK 17 module access.
| Surefire config already includes `--add-opens`; ensure your runner inherits it.

| Coverage below threshold
| Missing `*Test` / `*PropertyTest`.
| Add twin test files per AGENTS.md policy.

| `openapi-generator` fails
| Schema file missing.
| Verify `src/main/resources/schemas/openapi.yaml`.
|===
